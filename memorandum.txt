othelloプログラムについてメモを残す

- referee, player1, player2を3スレッドたてて動かす
(現時点ではbusy-waitになるので3CPUを占有してしまう)

- ロックを使わずに共有変数で乗り切る
(余裕があればロックによる実装に変更する)

- 通信するための整数値について

事前に値に対する意味を設定しておく。client側で適切に解釈を行う。

n = 0 : 初期手
0 <= n <= 99 : 手
0000 <= n <= 9999 : 相手と自分の石の数(勝利)
10000 <= n <= 19999 : 相手と自分の石の数(敗北)
20000 <= n <= 29999 : 相手と自分の石の数(引き分け)

n < 0 : エラー
